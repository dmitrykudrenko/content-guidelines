# Content Guidelines

Use this skill to apply the content guideline system in this repository. Keep the guidelines as the source of truth; do not invent product-specific rules when the repository marks them as TODO or draft.

## Workflow

1. Read [00-INDEX.md](00-INDEX.md) first to understand the guideline map.
2. Select only the files relevant to the user's task from the routing table below.
3. Apply the selected guidelines to the user's artifact or request.
4. If the task is a review, report concrete findings, missing evidence, and recommended edits. Keep unverified claims explicit.
5. If the task is content creation, produce a draft that satisfies the required content type, VI level, audience intent, and workflow stage.

## Defaults

- Treat files marked `DRAFT` or illustrative as usable guidance, not final company policy.
- Preserve the guideline vocabulary: Valuable Information, AVI, VI level, Note, Article, Scaling, Product, eBook.
- Prefer concrete examples, evidence, internal data, product mastery, and systematization over generic advice.
- For Yespo, Claspo, or Stripo work, adapt only what is supported by the provided artifact or selected guideline files; flag missing product-specific guidelines instead of filling gaps silently.
- For reviews, prioritize issues that affect content value, factual support, product accuracy, structure, CTA, SEO-value balance, and guideline fit.
